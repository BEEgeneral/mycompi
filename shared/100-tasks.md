# AGENTE 1: Strategy & Mission (Diana)
**Tareas: 1-6, 91-100 (22 tareas)**

## Substage 1.1: Plan Mission (Tasks 1-6)

### TASK 1: Define Goals
**Purpose:** Definir el North Star del cliente. Antes de comprometer recursos, alinear objetivos con la visión.
- Step 1: Listar top 3-5 objetivos estratégicos para los próximos 3 años
- Step 2: Cuantificar cada objetivo en OKRs/KPIs medibles
- Step 3: Mapear cómo cada área de negocio apoya estos objetivos
- Step 4: Crear statement de una página
- Step 5: Validar con liderazgo del cliente

**Quick Win:** Crear dashboard visual con top 5 goals y estado actual

### TASK 2: Gather All Steps
**Purpose:** Entender el camino completo de idea a lanzamiento.
- Step 1: Brainstorming con el equipo
- Step 2: Listar TODOS los pasos desde idea hasta lanzamiento
- Step 3: Agrupar en workstreams
- Step 4: Identificar dependencias
- Step 5: Estimar duración y recursos
- Step 6: Crear roadmap visual
- Step 7: Review con mentores

**Quick Win:** Spreadsheet con todas las tareas, owners y deadlines

### TASK 3: Streamline Steps
**Purpose:** Eliminar ineficiencias. El 30-40% del esfuerzo se desperdicia en actividades de bajo valor.
- Step 1: Documentar cada workflow principal
- Step 2: Analizar cada paso: valor vs tiempo
- Step 3: Identificar top 3 cuellos de botella
- Step 4: Rediseñar workflows
- Step 5: Implementar automatización
- Step 6: Crear SOPs
- Step 7: Train equipo

**Quick Win:** Auditar un proceso clave esta semana, identificar top 3 inefiencias

### TASK 4: Determine Budget to Fund Plan
**Purpose:** La mayoría de startups mueren por falta de caja. El budget es decisión estratégica.
- Step 1: Determinar budget total de corporate venturing para el año
- Step 2: Modelar necesidades de funding por etapa
- Step 3: Aplicar portfolio thinking
- Step 4: Gates basados en milestones
- Step 5: Reservar 10-20% contingencia
- Step 6: Documentar funding map
- Step 7: Obtener aprobación CFO/Board

**Quick Win:** Crear forecast de 12 meses con allocation venture-by-venture

### TASK 5: Allocate Budget to Launchpad and Ventures
**Purpose:** Balancear servicios centrales vs funding dedicado.
- Step 1: Listar ventures activos y su etapa
- Step 2: Calcular overhead del Launchpad
- Step 3: Estimar funding por venture por etapa
- Step 4: Aplicar weightings de portfolio
- Step 5: Ajustar hasta equilibrar
- Step 6: Documentar governance

**Quick Win:** Crear split de budget: % central vs ventures

### TASK 6: Set Financial Targets
**Purpose:** Traducir ambición en accountability. Definir qué es éxito.
- Step 1: Definir top 5 métricas financieras del Launchpad
- Step 2: Establecer targets a 3 años
- Step 3: Cascada a nivel venture individual
- Step 4: Leading indicators para cada target
- Step 5: Cadencia de reporting (mensual/trimestral)
- Step 6: Dashboard financiero
- Step 7: Review con CFO y board trimestral

**Quick Win:** One-page summary con top 5 métricas, estado actual, y meta 3 años

## Substage 1.4: Validate Ideas (Tasks 17-24)

### TASK 17: Distil into Short List
**Purpose:** Análisis parálisis destruye startups. Short list de 3-5 opciones permite deep analysis.
- Step 1: Definir screening criteria
- Step 2: Apply "kill criteria" primero
- Step 3: Score remaining models
- Step 4: Group similar models en clusters
- Step 5: Select representative model(s) de cada cluster
- Step 6: Create short list de 3-5 opciones
- Step 7: Document rationale

**Quick Win:** Apply kill criteria a tu long list e identificar top 5 para short list

### TASK 18: Compare How to Innovate for Short List
**Purpose:** Diferentes business models requieren diferentes innovation approaches.
- Step 1: Para cada modelo, identificar primary innovation type
- Step 2: Assess qué cada modelo necesita para succeed
- Step 3: Compare innovation requirements a internal capabilities
- Step 4: Score cada modelo en "innovation fit"
- Step 5: Identificar cuáles tienen best opportunity + capability
- Step 6: Document innovation approach

**Quick Win:** Para cada shortlisted model, identificar top 3 innovation requirements

### TASK 19: Compare How to Compete in Blue Ocean for Short List
**Purpose:** Blue Ocean strategy ayuda encontrar uncontested market space.
- Step 1: Para cada shortlisted model, crear strategy canvas
- Step 2: Plot current market offerings
- Step 3: Perform ERRRC analysis (Eliminate, Reduce, Raise, Create)
- Step 4: Calculate value-cost implications
- Step 5: Score Blue Ocean potential
- Step 6: Select modelo con strongest differentiation

**Quick Win:** Crear strategy canvas para top shortlisted model e identificar una ERRRC action

### TASK 20: Compare Using Business Model Canvas for Short List
**Purpose:** Business Model Canvas es herramienta poderosa para comparar side-by-side.
- Step 1: Para cada shortlisted model, completar los 9 canvas blocks
- Step 2: Compare key blocks (especialmente Value Prop y Revenue Model)
- Step 3: Identificar critical dependencies y assumptions
- Step 4: Run "what if" scenarios
- Step 5: Score cada modelo en desirability, feasibility, viability
- Step 6: Select best overall fit

**Quick Win:** Completar Business Model Canvas para cada shortlisted model y comparar top 3 blocks

### TASK 21: Compare Using Customer Discovery
**Purpose:** Validación última: ¿los clientes comprarán?
- Step 1: Para cada shortlisted model, document customer hypotheses
- Step 2: Create interview guide
- Step 3: Conduct 10-15 customer interviews por modelo
- Step 4: Analyze responses para patterns
- Step 5: Map findings a business model canvas
- Step 6: Score cada modelo en customer validation strength
- Step 7: Select modelo con strongest signals

**Quick Win:** Conduct 3 customer interviews para tu top model y documentar insights

### TASK 22: Rank Business Models on Short List
**Purpose:** Ranking requiere hacer trade-offs hard explícitos.
- Step 1: Definir ranking criteria y weights
- Step 2: Score cada modelo en cada criterion (1-5)
- Step 3: Calculate weighted scores
- Step 4: Run sensitivity analysis
- Step 5: Conduct reference conversations
- Step 6: Make final selection
- Step 7: Document rationale

**Quick Win:** Crear scoring matrix para shortlisted models y calculate weighted scores

### TASK 23: Build Proof of Concept on #1 Business Model
**Purpose:** PoC testa core assumption con minimal investment.
- Step 1: Identificar #1 business model de ranking
- Step 2: Extract core hypothesis para testear
- Step 3: Design minimum viable experiment
- Step 4: Define success criteria upfront
- Step 5: Execute PoC within time box (4-8 semanas)
- Step 6: Analyze results
- Step 7: Document learnings

**Quick Win:** Definir top 3 risky assumptions de tu #1 model y diseñar test para riskiest

### TASK 24: Follow Lean Startup Loop Until Customer Validation
**Purpose:** Build-Measure-Learn loop para validated learning eficiente.
- Step 1: Build MVP
- Step 2: Get MVP in front of target customers
- Step 3: Measure - track key metrics
- Step 4: Learn - analyze data vs assumptions
- Step 5: Pivot or persevere
- Step 6: Repeat until metrics hit predefined thresholds
- Step 7: Customer validation achieved

**Quick Win:** Definir Build-Measure-Learn loop para tu current experiment y track weekly
