### Carlos — Ventas y CRM

**Proposal 2:**
- **Título:** Micro-segmentación con señales de intención para MyCompi
- **Trend/Fuente:** Pharow Barometer Outbound 2026 + HubSpot Sales Sequence Guide (Feb-Mar 2026)
- **Viabilidad:** 🟢 Alta
- **Esfuerzo:** bajo-medio
- **Expected Impact:** +30-40% response rates en cold outreach vs targeting genérico
- **Resumen:** En lugar de emailing masivo a PYMES españolas genéricas, Carlos debería micro-segmentar usando señales concretas: empresas que han contratado récemment (ofertas de empleo en LinkedIn), empresas con actividad digital reciente, o que han instalado herramientas de automatización (detectable via su web). Esto permite ángulos de outreach mucho más personalizados y relevantes.
- **Datos de soporte:** Pharow: micro-segmentación con señales (recruiting, levée de fonds, nouveaux outils) = tasas de respuesta significativamente más altas. HubSpot: sequences más cortas (2-4 emails) con targeting preciso > sequences largas genéricas. En MyCompi contexto: 49€/mes, target PYMES 5-50 empleados → señales como 'hashtag automation' en su web, actividad LinkedIn, o招聘信息 son proxies de necesidad real.
- **Próximo paso:** Configurar alertos en Pharow o similar para detectar empresas españolas que contratan perfiles marketing/digital y tienen 10-50 empleados.

---

**Proposal 3:**
- **Título:** Sequencias de 3-4 touchpoints multi-canal (email + LinkedIn) para MyCompi
- **Trend/Fuente:** HubSpot Sales Sequence Guide 2026 + Pharow Barometer
- **Viabilidad:** 🟢 Alta
- **Esfuerzo:** medio
- **Expected Impact:** +20% reply rates vs email solo (dato Pharow: omnichannel 3x mejor que single-channel)
- **Resumen:** El outbound actual de MyCompi es solo email. Añadir LinkedIn connection request al día 1 y LinkedIn message al día 3-4 (después del primer email) es低成本, high impact. No requiere herramientas caras: LinkedIn Sales Navigator básico + Apollo para emails basta para empezar.
- **Datos de soporte:** Pharow: multicanal (email + LinkedIn) supera 3x en conversión vs single-channel. HubSpot: mejores secuencias son 2-4 mensajes máximo, spacing 2-3 días, cada touchpoint con valor diferente. No 'break-up emails' en mercados de bajo volumen.
- **Sample sequence para MyCompi:** Día 1: Email personalizado (max 75 palabras, ángulo: problema de overload de tareas en PYMEs). Día 2: LinkedIn connection request (note corta referencia al email). Día 4: Email follow-up con caso de uso concreto. Día 7: LinkedIn message si no hay respuesta. Fin: 'Should I close your file?' approach.

---

**Proposal 4:**
- **Título:** Herramienta de sales automation — Instantly.ai para warmup y envío
- **Trend/Fuente:** Comparativas Instantly vs Smartlead vs Apollo (Enero 2026, G2 reviews)
- **Viabilidad:** 🟢 Alta
- **Esfuerzo:** bajo
- **Expected Impact:** Mejor entregabilidad de emails, menos riesgo de blacklisting de dominio
- **Resumen:** Para el outbound de MyCompi, Instantly.ai es la opción más sencilla y económica para cold email. Incluyen warmup automático de dominios (crítico para no quemar dominios), pricing flat fee (no per-seat), y integración directa con Apollo/LinkedIn. Alternativa: Smartlead si se necesita multi-dominio. Apollo.io v3 tiene base de datos B2B española buena pero es más complejo.
- **Datos de soporte:** Instantly rating 4.8/5 en G2, flat fee pricing, warmup infrastructure incluido. Smartlead: mejor para equipos que necesitan más granular control de warmup. Apollo: mejor base de datos pero más caro y complejo.
- **Recomendación para MyCompi:** Empezar con Instantly + Apollo (hero tier ~80$/mes combinado) o Smartlead + Kaspr para LinkedIn data enrichment.

### Marcos — Desarrollo Web

**Proposal 1: AI-assisted code review tooling**
- **Trend/Fuente:** LogRocket + EU Code Week — "vibe coding" y AI explicando codebases en 2026
- **Viabilidad:** 🟢 Alta
- **Esfuerzo:** bajo
- **Expected Impact:** Reducir deuda técnica, onboarding más rápido de nuevos features
- **Resumen:** Integrar tooling de AI para explanations del código existente. En MyCompi, con 7 agentes trabajando sobre el mismo codebase, mantener coherencia es clave.
- **Technical notes:** GitHub Copilot Chat o Claude for VSCode en el entorno de desarrollo; no requiere cambios en producción

---

**Proposal 2: Edge computing para la landing**
- **Trend/Fuente:** Figma Web Dev Trends 2026 — "server-first performance"
- **Viabilidad:** 🟡 Media
- **Esfuerzo:** alto
- **Expected Impact:** Mejora measurable de Core Web Vitals (LCP), mejor SEO ranking
- **Resumen:** Migrar landing a CDN edge (Cloudflare Pages o similar) desde Render actual. Reducción latencia global ~40%.
- **Technical notes:** Requiere cambio de hosting o añadir Cloudflare como proxy. Evaluar si merece el esfuerzo dado que el tráfico actual es bajo.

---

**Proposal 3: Schema markup FAQ + HowTo para SEO**
- **Trend/Fuente:** SEO trends 2026 — voz search, snippet acquisition, structured data
- **Viabilidad:** 🟢 Alta
- **Esfuerzo:** bajo
- **Expected Impact:** Captura de featured snippets, mejora en voice search results
- **Resumen:** La landing ya tiene sección FAQ. Añadir schema.org/FAQPage markup es trivial (1-2h) y tiene impacto directo en SEO.
- **Technical notes:** Añadir JSON-LD en el `<head>` de la landing. Solo requiere editar el HTML build output o el componente React.
